# The **Perceptron**

The **Perceptron** is a type of artificial neural network invented in 1957.

- It can be seen as the basic unit of a more complex neural network.
    
- It is a simplified representation of a biological neuron and consists of a supervised learning algorithm for binary classification, which means it can predict whether an input belongs to one of two possible classes.
    
- Basic structure of a **Perceptron**:
- 
![Perceptron Structure](shared-utils/img_deep_learning_01.png)

**Note:** Each added layer is purely mathematical; the overall structure is a set of mathematical operations.

- **Input (Inputs)**
    
- **Weights:** Representation of the relationship between X and Y.
    
- **Weighted Sum**
    
- **Activation Function:** Handles the linearity of the data.
    
- **Output:** Model prediction compared with the real value.
    
- **Weight Update:** Backpropagation algorithm.

---

## What the Perceptron Can Solve

It is capable of solving linearly separable problems, meaning when two
classes can be separated by a straight line (or, in higher dimensions, a
plane or hyperplane).

---
## Importance of the Perceptron

It provided the foundation for the development of more complex neural
networks, significantly contributing to the field of machine learning.

### Multilayer Neural Networks (MLP - MultiLayer Perceptron)

A significant evolution in Artificial Intelligence and machine learning,
marked by advances, challenges, and a period known as the "AI winter."

In the 1980s, researchers began exploring multi-layer neural network
models that could learn more complex representations of data.

The introduction of the backpropagation algorithm allowed multi-layer
networks to effectively adjust their internal weights and learn to
represent non-linear functions.

---
### Contributions of MLPs

-   Backpropagation provided a way for neural networks to learn from
    errors, adjusting weights not only in the output layer but also in
    hidden layers.
-   MLPs are composed of an input layer, one or more hidden layers, and
    an output layer, with each layer containing multiple neurons.
-   Each neuron in the hidden layers can model non-linear interactions
    between inputs, and adding multiple hidden layers enables the
    network to learn increasingly complex representations.

# Convolutional Neural Networks (CNNs) and Recurrent Neural Networks (RNNs)

These are two fundamental deep neural network architectures that excel in different types of machine learning tasks.

## CNNs

- Effective in processing data with a spatial grid structure, such as images, where the relative location of features is essential for tasks like classification and object detection.
    
- Characterized by convolutional layers that apply filters to input data, capturing local features. These filters are small in width and height but extend across the full depth of the input volume.
    
- After each convolutional layer, there is typically a pooling layer that reduces spatial dimensions and aggregates important features.
    
- At the end of a CNN, fully connected layers interpret the high-level features extracted by convolution and pooling layers to make predictions or classifications.


![[img_deep_learning_06.png]]

## RNNs

- Especially suitable for sequential data, such as natural language or time series, where order and context matter.
    
- The distinctive property of RNNs is their ability to maintain a state or memory over time.
    
- This allows RNNs to form a kind of memory about previous elements in a sequence, which is crucial for understanding context and managing predictions.
    
- However, standard RNN architectures often struggle to learn long-term dependencies due to problems such as vanishing or exploding gradients.
    
To overcome these issues, variations such as LSTM (Long Short-Term Memory) and GRU (Gated Recurrent Units) were developed.

![[img_deep_learning_07.png]]

While CNNs impose a local hierarchical structure for pattern recognition, RNNs address the sequential and temporal nature of data.

# Optimization and Regularization

Optimization in Deep Learning refers to the process of adjusting model parameters — typically neural connection weights — to minimize a loss function, which measures the difference between model predictions and true expected outcomes. The goal is to find the optimal set of parameters that results in the best possible model performance on training data.

Regularization consists of techniques aimed at preventing overfitting and promoting model generalization. This can be achieved in several ways, such as penalizing model complexity, injecting noise during training, or artificially increasing the size of the training dataset (data augmentation).

One of the most common regularization methods is adding regularization terms to the loss function using L1 and L2 norms of the weights, discouraging large weights and potentially leading to a simpler, less overfitted model.

Other regularization techniques include:

- **Dropout:** Randomly disabling neurons during training to prevent the model from relying too heavily on any specific path within the network.
    
- **Early Stopping:** Halting training as soon as performance on a validation set begins to degrade, thus preventing overtraining.
    

**In summary:** Optimization adjusts the model to fit the data, while regularization ensures the model generalizes well beyond the training dataset.

---
# Dropout and Batch Normalization

These are regularization and optimization techniques, respectively, used in Deep Learning to improve training and generalization of deep neural networks.

Dropout acts more directly to prevent overfitting by forcing the network to learn more robust representations, while Batch Normalization helps accelerate training and stabilize learning in very deep networks.
